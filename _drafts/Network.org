* Proxy
** How to config primary DNS
- Solution 1: configure network connection
  [Network Connections]
=>[ethernet or wifi interface]
=>[IPv4 Setting]
=>change Method from 'Automatic (DHCP)' to 'Automatic (DHCP) Address Only'

Edit DNS server: 8.8.8.8
- Solution 2: modify /etc/network/interfaces
#+BEGIN_SRC sh
dns-nameservers 8.8.8.8
#+END_SRC
- Solution 3: modify /etc/resolvconf/resolv.conf.d/base
#+BEGIN_SRC sh
nameserver 8.8.8.8
nameserver 8.8.4.4
#+END_SRC
then
#+BEGIN_SRC sh
resolvconf -u
#+END_SRC
- verify DNS config
#+BEGIN_SRC sh
cat /etc/resolv.conf

yubao@yubao-Lenovo-YOGA-910-13IKB:~$ cat /etc/resolv.conf 
# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
# 127.0.0.53 is the systemd-resolved stub resolver.
# run "systemd-resolve --status" to see details about the actual nameservers.

nameserver 8.8.8.8
nameserver 127.0.0.53
#+END_SRC
Restart you computer if not work.

nslookup www.youtube.com
#+BEGIN_SRC sh
yubao@yubao-Lenovo-YOGA-910-13IKB:~$ nslookup www.youtube.com  
Server:		8.8.8.8
Address:	8.8.8.8#53

Non-authoritative answer:
Name:	www.youtube.com
Address: 8.7.198.45
#+END_SRC

If failed, try 'nslookup www.youtube.com 8.8.8.8', if succeed, retry to configure the DNS again

ping www.youtube.com

** Shadowsocks
*** How to deploy SS in ubuntu
- Install
#+BEGIN_SRC sh
 apt-get update
 apt-get install python-pip
 pip install shadowsocks
#+END_SRC
- Write JSON configure file
#+BEGIN_SRC javascript
 {
     "server":"x.x.x.x",
     "server_port":3303,
     "local_address":"127.0.0.1",
     "local_port":1080,
     "password":"xxxxxxxx",
     "timeout":300,
     "method":"aes-256-cfb",
     "fast_open": false,
     "workers": 1
 }
#+END_SRC
- run in shell
#+BEGIN_SRC sh
sslocal -c <your_config_file>.json
#+END_SRC
run in backgroud
#+BEGIN_SRC sh
nohup sslocal -c <your_config_file>.json $
#+END_SRC

- configure proxy in Web browser
port: 1080
server: 127.0.0.1
ss: socks5
** Common Problems
*** DNS problems 
- Q: VPN or Shadowsocks has been connected, however I cannot access www.youtube.com via web browser?

Configure DNS: 
The primary dns should be set to 8.8.8.8 or 8.8.4.4, rather than the local address, like 127.*, 198.*. Therefore, we must to configure the Primary DNS.
#+BEGIN_SRC  sh
yubao@yubao-Lenovo-YOGA-910-13IKB:~$ nslookup
> www.youtube.com
Server:		127.0.0.53
Address:	127.0.0.53#53

Non-authoritative answer:
Name:	www.youtube.com
Address: 8.7.198.45

#+END_SRC

- IP forwarding is not opened

#+BEGIN_SRC sh
emacs -nw /etc/sysctl.conf
uncomment:
net.ipv4.ip_forward=1
#+END_SRC
