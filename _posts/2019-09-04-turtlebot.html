3 Network Configuration

Config Turtlebot robot as the master machine. My PC as the slave machine.

  1. Install ssh
    sudo apt-get install openssh-server
ps -e|grep ssh

ssh [user_name]@[ROBOT_IP]
  2. Network Time Protocol
    sudo apt-get install chrony
sudo ntpdate ntp.ubuntu.com

ntpdate -q other_computer_ip
  3. Add host name
    sudo vim /etc/hosts

    Add: 

    133.15.23.26    yubao-pc
133.15.23.206   kobuki

    Restart Network 

    sudo /etc/init.d/networking restart
  4. Modify bash file to config ROS

    Kobuki: 

    echo export ROS_MASTER_URI=http://IP_OF_TURTLEBOT:11311 >> ~/turtlebot/devel/setup.sh
echo export ROS_HOSTNAME=IP_OF_TURTLEBOT >> ~/turtlebot/devel/setup.sh

    Remote PC Setup: 

    echo export ROS_MASTER_URI=http://IP_OF_TURTLEBOT:11311 >> ~/.bashrc
echo export ROS_HOSTNAME=IP_OF_PC >> ~/.bashrc

    or add these lines to ~/.bashrc 

    export ROS_MASTER_URI=http://[kobuki-ip]:11311
export ROS_HOSTNAME=<PC_IP>
export ROS_IP=<PC_IP>

    Example: 

    export ROS_MASTER_URI=http://kobuki:11311
export ROS_HOSTNAME=133.15.23.26
export ROS_IP=133.15.23.26

12 pocketsphinx: ksm (keyword spotting mode)

Refer:

  • https://github.com/Pankaj-Baranwal/pocketsphinx/wiki/Demo:-Kws-mode
  • Pocketsphinx in ROS: Demo 1.0
  • build pocketsphinx ros wrapper
cd ~/catkin_ws/src
git clone https://github.com/Pankaj-Baranwal/pocketsphinx.git
cd ../..
catkin_make
  • Launch turtlebot_gazebo:
roslaunch turtlebot_gazebo turtlebot_world.launch
  • kws.launch
roslaunch pocketsphinx kws.launch dict:=/home/yubao/data/catkin_ws/src/pocketsphinx/demo/voice_cmd.dic kws:=/home/yubao/data/catkin_ws/src/pocketsphinx/demo/voice_cmd.kwlist
  • Checkout data
rostopic echo /kws_data
  • control robot

The turtlebot simulation subscribes to a topic: /mobile_base/commands/velocity. Our node publishes the required velocity and direction instructions on this topic which are read by gazebo and the turtlebot moves accordingly. 

rosrun pocketsphinx voice_control_example.py

dict: Location(absolute path) of the dictionary. Dictionary contains the words which are used in the keyphrases along with their phonetics as used in the acoustic model.

kwlist: This contains a list of keyphrases along with their threshold values. A typical line looks like this:

Hello World /1e-12/

keyphrase: In case you only need to use 1 keyphrase, you can simply use a combination of a keyphrase and its threshold(see next argument). You should either use a kwlist argument or a combination of keyphrase and threshold.

threshold: To specify the threshold of the keyphrase in the above mentioned argument.

input: If you want to use an audio file as input, you can specify its absolute path here. Else you should omit this argument. System's microphone will be used as input instead.

hmm: In case the default acoustic model is missing or not suited to your needs, you can specify a new one using this argument. Just add its absolute path as value of this argument. You can find the existing ones
